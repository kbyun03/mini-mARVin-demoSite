Localization

Localization is achieved by utilizing the heading data from the IMU and measurements from the LiDAR. Based on the heading, two walls are chosen, and the servo turns the LiDAR to face those walls. The servo has a range of 180 degrees therefore in most situations it can only face two walls. The LiDAR then takes a measurement of the distance to the two walls. Based on which walls are hit the LiDAR calculates the x and y position. For example, if the distance to wall 4 is chosen then the x coordinate is 77cm â€“ the distance. This localization approach is constrained by the accuracy of the IMU. If the heading angle from the IMU is very inaccurate than the coordinates calculated will be off significantly.

IMU

IMU is set up as following.
Enable I2C instructions Added two lines : I2c-dev ; I2c-bcm2708. Then update /boot/config.text Add to the bottom: dtparam =i2c_arm = on ; dtparam = i2c1 = on
Reboot, Connect I2C components, check for component address with i2cdetect
Based on I2C address from step above, change address in LSM9DS0.py

GUI

There are two parts to GUI : one window setting TCP communication between mini-mARVin and showing mini-mARVin's location in environment. GUI is developed using pyQT5 frameworks.
