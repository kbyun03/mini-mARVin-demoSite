Project Concepts

The goal of our project was to create a wirelessly-controlled miniature tactical unmanned ground vehicle (TUGV), which can can determine its own location, as well as obstacle locations, in its environment. These locations can then be plotted on a laptop-based graphical user interface (GUI). After encountering accuracy issues with the IMU and sonar sensors, we were unable to complete all of our goals. However, we did accomplish relative localization of mini-mARVin within an enclosed environment using IMU heading and lidar. The location of mini-mARVin was successfully plotted on the GUI interface after being sent to the laptop over WiFi. Additionally, keyboard control of mini-mARVin was enabled through the GUI. mini-mARVin's Functionalities

Localization

Determine mini-mARVin's position in the environment by using lidar and IMU sensors

Communication

Reliable TCP communication between mini-mARVin and computer station for data transfer and motor control

Motor Control

Wirelessly control mini-mARVin with keyboard inputs through the GUI

GUI Development

Graphical user interface developed so users can easily connect to mini-mARVin and navigate the environment with visual queues

Localization

Localization is achieved by utilizing the heading data from the IMU and measurements from the LiDAR. Based on the heading, two walls are chosen, and the servo turns the LiDAR to face those walls. The servo has a range of 180 degrees therefore in most situations it can only face two walls. The LiDAR then takes a measurement of the distance to the two walls. Based on which walls are hit the LiDAR calculates the x and y position. For example, if the distance to wall 4 is chosen then the x coordinate is 77cm – the distance. This localization approach is constrained by the accuracy of the IMU. If the heading angle from the IMU is very inaccurate than the coordinates calculated will be off significantly.

IMU

IMU is set up as following.
Enable I2C instructions Added two lines : I2c-dev ; I2c-bcm2708. Then update /boot/config.text Add to the bottom: dtparam =i2c_arm = on ; dtparam = i2c1 = on
Reboot, Connect I2C components, check for component address with i2cdetect
Based on I2C address from step above, change address in LSM9DS0.py

GUI

There are two parts to the GUI: one window begin TCP communication with mini-mARVin; the second showing mini-mARVin's location in environment. The GUI was developed using the pyQT5 libraries.

Motor Control

Motor control was implemented using an h-bridge motor driver, Tamiya Dual Gearbox motors, and WiFi communication with the GUI. Through the GUI, the user can control mini-mARVin using 'w', 's', 'a', 'd', and 'q' key presses for forward, back, left, right, and stop, respectively.

Parts List

Hardware Components: Raspberry Pi 3, 4 ultrasonic sensors (optional), TFMini LiDAR, HS-422HD servo, H-Bridge, LSM9DS0 IMU, Tamiya Dual Gearbox Motors, battery pack with DC barrel jack adaptor to power motors, USB power bank to power the RPi, and a wheeled vehicle body ready accomodate the necessary parts

Software Components: Python 3 libraries, pyQT5 libraries, and gpiozero raspberry pi library
